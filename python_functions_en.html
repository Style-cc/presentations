Functions are the building blocks of every program. Write once, reuse anywhere. Code becomes cleaner, shorter, and easier to read.

01Basics

What is a Function?

A function is a named block of code that only runs when called. It receives data, processes it, and returns a result.

♻️
Reusability
Write once — call as many times as needed, from anywhere
📦
Encapsulation
Hides implementation details. Only the input and output are visible from the outside
📖
Readability
Code with functions reads like plain text: calculate_tax(), send_email()
🐛
Debugging
Easier to find and fix a bug inside an isolated function
1# Define (once)
2def greet(name):
3    return f"Hello, {name}!"
4
5# Call (as many times as you like)
6greet("Alice") # → "Hello, Alice!"
7greet("Bob") # → "Hello, Bob!"

03Arguments

Types of Arguments

Python supports several types of arguments — from required positional to an arbitrary number.

Positional — required, passed in order
1def add(a, b):
2    return a + b
3add(3, 4) # → 7
Default — used when not provided
1def power(n, exp=2):
2    return n ** exp
3power(3) # → 9 (exp=2)
4power(2, 10) # → 1024
*args — any number of arguments
1def total(*nums):
2    return sum(nums)
3total(1,2,3,4) # → 10
4# nums = (1, 2, 3, 4)
**kwargs — keyword arguments
1def info(**data):
2    for k,v in data.items():
3        print(k, v)

04return

The return Statement

return exits the function and sends a value back to the caller. A function without return returns None.

return variants
1# Return a single value
2def square(x):
3    return x ** 2
4
5# Return multiple values (tuple)
6def min_max(lst):
7    return min(lst), max(lst)
8
9lo, hi = min_max([3,1,5])
10# lo=1, hi=5
Early exit
1# return exits the function immediately
2def find_first_even(lst):
3    for n in lst:
4        if n % 2 == 0:
5            return n # exits immediately
6    return None # nothing found
7
8find_first_even([1,3,4,6]) # → 4
💡 None
If a function does not explicitly return anything — Python automatically returns None.
05Scope

Scope — scope

Variables inside a function only exist there. They are not accessible from outside. This is called local scope.

🌍 Global scope
x = 10
name = "World"
🏠 Local (inside function)
y = 20 # local only
result = x + y # x is visible!
1x = 10 # global
2
3def my_func():
4    y = 20 # local
5    print(x) # ✓ visible from outside
6    print(y) # ✓ local variable
7
8print(y) # ✗ NameError!
9
10# global — modify the global variable
11def inc():

06Lambda

Lambda Functions

An anonymous one-line function. No name, no code block. Used where a function is needed just once — typically as an argument.

Syntax
1# lambda arguments: expression
2square = lambda x: x ** 2
3square(5) # → 25
4
5# Equivalent to:
6def square(x):
7    return x ** 2
8
9# Multiple arguments
10add = lambda a, b: a + b
Where to use — with sort, map, filter
1words = ["banana", "apple", "fig"]
2
3# Sort by word length
4words.sort(key=lambda w: len(w))
5# → ['fig', 'apple', 'banana']
6
7# map — apply to each element
8list(map(lambda x: x*2, [1,2,3]))
9# → [2, 4, 6]
10
11# filter — filter elements
12list(filter(lambda x: x%2==0, [1,2,3,4]))
13# → [2, 4]
07Documentation

Docstrings & Type Hints

Good code explains itself. A docstring describes what a function does. Type hints annotate the argument and return value types.

Docstring
1def divide(a, b):
2    """
3    Divides a by b.
4    Args: a (int), b (int)
5    Returns: float
6    Raises: ZeroDivisionError
7    """
8    if b == 0:
9        raise ValueError("b cannot be 0")
10    return a / b
Type hints — Python 3.5+
1# Arguments: annotate type with :
2# # Return type: after ->
3def greet(name: str) -> str:
4    return f"Hello, {name}!"
5
6def add(a: int, b: int) -> int:
7    return a + b
8
9def items(lst: list) -> None:
10    for x in lst: print(x)
💡 Type hints are optional
Python does not enforce types at runtime. Hints are documentation for the developer and tooling (IDE, mypy).

09Summary

Summary: what we learned

Key concepts from this lesson

def
Define a function with the def keyword. Body is an indented block. Call it with its name and parentheses.
Arguments
Positional, default, *args (tuple), **kwargs (dict) — four types.
return
Returns a value. You can return multiple values as a tuple. No return → None.
Scope
Local variables are only visible inside the function. global allows modifying a global.
lambda
Anonymous one-line function. Perfect for sort(key=), map(), filter().
